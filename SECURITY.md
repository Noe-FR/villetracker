# Politique de sécurité

## Signaler une vulnérabilité

Si vous découvrez une faille de sécurité dans ce projet, merci de la signaler de manière responsable.

**Créez une issue GitHub** avec le label `security` en incluant :

- Une description claire de la vulnérabilité
- Les étapes pour la reproduire
- L'impact potentiel estimé
- Si possible, une suggestion de correction

> Pour les vulnérabilités sensibles (accès à des données privées, contournement d'authentification), vous pouvez contacter directement via le profil GitHub avant de publier l'issue publiquement.

## Ce qui est dans le périmètre

- Le code frontend React de ce dépôt
- Les interactions avec l'API publique

## Ce qui est hors périmètre

- Le backend et l'API (dépôt séparé)
- Les données sources (data.gouv.fr, INSEE, etc.)
- L'infrastructure d'hébergement

## Délai de réponse

Les issues de sécurité sont traitées en priorité. Une première réponse sera apportée sous **72 heures**.
